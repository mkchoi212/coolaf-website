Transducers in Swift — Part II

Let’s get the show started right away.

Before we get started, let us declare some operators that will make function composition a little easier to deal with.

Function Composition Operator

infix operator |> {associativity left}
func |> <A, B>(x: A, f: A -> B) -> B {
    return f(x)
}
func |> <A, B, C>(f: A -> B, g: B -> C) -> A -> C{
    return f(x)
}

Now we can do stuff like

// Instead of abs(sqrt(incr(-3)))
-3 |> incr |> sqrt |> abs

This kind of looks like Unix pipelines, right? And if this guy said pipelines are a good thing, than this must be good! 

Now, let’s add a mapping function to do what we just did for an array of data.

Basic Curried Functions

func fmap <A, B>(f: A -> B) -> [A] -> [B] {
    return { map($0, f) }
}

This simply takes a A -> B function and makes a function that [A] -> [B] . Now we can do stuff like

userData |> fmap(parse) |> fmap(getFavs)
